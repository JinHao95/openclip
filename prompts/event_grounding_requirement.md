# 事件精确定位（Event Grounding）

## 任务

给定一个已识别的足球比赛事件及其周围的转录文本，精确定位最佳短视频剪辑的起止时间。

## 输入

- **事件标题**：{title}
- **事件类型**：{event_type}
- **锚点时间**：{anchor_time}
- **重要性**：{importance}

## 转录文本

以下是锚点时间前后的 SRT 转录内容：

{srt_window}

## 定位规则

### 通用规则：
- start_time 和 end_time 必须是转录中存在的实际时间戳
- 严禁编造时间戳
- 格式 HH:MM:SS

### 按事件类型定位：

| event_type | start_time 选择 | end_time 选择 |
|-----------|----------------|--------------|
| goal | 最后一脚关键传球/起脚射门（含 3-5 秒铺垫）| 进球确认 + 第一句即时反应（不含回放分析）|
| shot | 进攻组织开始（最后 1-2 次传球）| 射门结果确认（偏出/被扑/中柱）|
| save | 射门发起 | 扑救完成 + 解说确认 |
| card | 犯规动作开始 | 裁判出示牌 + 第一句反应 |
| var | VAR 介入信号 | 最终判罚结果确认 |
| tactical | 战术讨论第一句 | 该段分析自然结束处 |
| analysis | 观点开始 | 观点结束（在话题转换前）|
| celebration | 进球/事件确认后 | 庆祝结束或转播切走 |
| substitution | 换人提示 | 换人完成 + 简短评价 |
| other | 事件相关的第一句 | 事件结束的自然停顿处 |

### 时长约束：
- 最终 duration 必须在 {min_duration} ~ {max_duration} 秒之间
- 如果自然边界超出范围，优先保证核心动作完整，适当缩减铺垫或后续

### 避免：
- 包含事件结束后的无关内容（回放分析、广告等）
- 在句子中间截断
- start_time 早于锚点 90 秒以上
- end_time 晚于锚点 90 秒以上

## 输出格式

只返回 JSON，不要附加解释：

```json
{
  "start_time": "HH:MM:SS",
  "end_time": "HH:MM:SS",
  "duration_seconds": N
}
```
