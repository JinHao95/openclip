# 足球赛事高光分析

## 任务

分析提供的足球比赛转录文本，识别值得制作短视频切片的精彩时刻。

**关键约束**：只分析提供的转录文本。所有时间戳必须存在于实际转录中——严禁编造时间戳。

## 内容类型

本分析固定为 `sports`（足球赛事）类型。输出 `detected_content_type` 字段统一填 `"sports"`。

## 识别标准

### 高优先级（必须捕获）：
- 进球（含点球、任意球、乌龙球）
- 红牌/黄牌/重大犯规
- 关键扑救（门将神扑）
- 比分变化瞬间
- 绝杀/逆转/扳平

### 中优先级：
- 精彩射门（中柱/门框/被扑）
- 战术亮点（经典配合/阵型变换）
- 球星个人表现
- VAR 争议判罚
- 关键换人/战术调整

### 低优先级（仍应标注）：
- 解说精彩点评/金句
- 历史纪录/数据对比
- 赛前分析/赛后总结
- 教练反应/球员庆祝

## 数量要求（重要）

- 识别所有有明显动作的时刻（射门、扑救、进球、犯规、庆祝、战术配合等）
- 不要过度筛选——中等质量的时刻也要包含
- 后续聚合步骤会做最终筛选，所以宁多勿少
- 如果转录中有动作内容，至少返回 1 个时刻

## 锚点时间确定原则（关键）

**必须使用转录中的实际时间戳**
- 严禁编造或猜测时间戳
- anchor_time 必须是事件核心发生瞬间对应的转录时间戳
- **注意**：`00:01:55` 表示 1 分 55 秒，不是 `01:55:00`（1 小时 55 分）

**anchor_time 确定方法：**
- 进球：射门/头球等动作发生的那一句话的开始时间（不是进球后的分析或回放）
- 扑救：门将扑出的那一句描述
- 射门：射门动作发生时
- 战术/分析类：核心观点出现时
- 如果有多句相关，选最核心的动作发生句

## 重叠处理
- 如两个时刻 anchor_time 非常接近（< 10秒），保留更强的那个
- 不要对同一个进球事件标注多个 anchor

## 输出格式

返回 JSON，严格遵循以下结构：

```json
{
  "video_part": "chunk001",
  "detected_content_type": "sports",
  "engaging_moments": [
    {
      "title": "凯恩点球破门！英格兰1-0领先克罗地亚",
      "anchor_time": "00:20:25",
      "event_type": "goal",
      "importance": "high",
      "summary": "凯恩主罚点球命中，英格兰率先取得领先。",
      "why_engaging": "本场首粒进球，比分改写，凯恩展现巨星本色",
      "tags": ["进球", "点球", "明星球员", "赛事高光"]
    },
    {
      "title": "索斯盖特围绕凯恩打造拜仁体系",
      "anchor_time": "00:22:20",
      "event_type": "analysis",
      "importance": "medium",
      "summary": "解说分析英格兰战术体系如何围绕凯恩回撤展开。",
      "why_engaging": "深入拆解战术，揭示凯恩回撤与拜仁踢法的关联",
      "tags": ["战术解析", "球星表现", "深度解析"]
    }
  ],
  "total_moments": 2,
  "analysis_timestamp": "2024-01-01T12:00:00Z"
}
```

**时间戳关键规则：**
1. anchor_time 必须对应转录中的实际时间戳
2. 不要使用占位符或示例时间戳
3. 无法找到有效时间戳时返回空数组
4. 格式为 HH:MM:SS，不要 SRT 毫秒格式

**quality要求：**
1. 可识别多个时刻
2. 如无时刻满足标准，返回 `"engaging_moments": []` + `"total_moments": 0`
3. 不要强行标注低质量内容
4. 宁可返回零时刻，也不要返回低质量或编造的内容

## 字段说明

### 顶层必填字段：
- **video_part**: 视频片段标识（如 "chunk001"）
- **detected_content_type**: 固定为 "sports"

### 每个时刻必填字段：
- **title**: 15-25 个汉字，必须包含球员名 + 动作结果（遵循标题撰写指南）
- **anchor_time**: HH:MM:SS 格式，事件核心发生瞬间
- **event_type**: 事件类型（见下表）
- **importance**: "high" / "medium" / "low"
- **summary**: 1-2 句话描述发生了什么
- **why_engaging**: 为什么这个时刻值得做成短视频
- **tags**: 从已批准标签中选 3-5 个

### event_type 枚举：
| event_type | 使用场景 |
|-----------|---------|
| goal | 进球（含点球、任意球、乌龙球） |
| shot | 射门未进（中柱/被扑/偏出） |
| save | 门将扑救 |
| card | 红牌/黄牌 |
| foul | 重大犯规 |
| var | VAR 判罚 |
| substitution | 换人 |
| celebration | 庆祝 |
| tactical | 战术亮点/阵型变化 |
| analysis | 解说分析/点评/金句 |
| other | 其他 |

**event_type 优先级规则（关键）：**
当一个片段同时涉及多种内容时，按以下优先级选择 event_type：
**goal > shot > save > card > foul > var > tactical > analysis**
例如：解说一边回放一边分析凯恩的跑位如何完成进球 → event_type = goal（不是 analysis），战术分析内容通过 tags 体现

### importance 判定：
- **"high"**: 进球、绝杀、红牌、关键扑救等改变比赛走向的时刻
- **"medium"**: 精彩射门、战术亮点、有价值的解说分析
- **"low"**: 一般性战术讨论、数据回顾、赛况描述

## 已批准标签

### 足球动作（描述场上发生了什么）：
["进球", "射门", "头球", "点球", "红牌", "黄牌", "扑救", "任意球", "角球", "越位", "VAR", "换人", "助攻", "反击", "庆祝", "绝杀", "逆转", "乌龙球", "明星球员", "战术亮点", "赛事高光"]

### 足球分析（描述解说在做什么类型的点评）：
["战术解析", "球星表现", "历史纪录", "裁判争议", "数据分析", "教练布置", "阵型变化", "体能管理", "赛前分析", "赛后总结", "经典对决", "名嘴金句"]

### 通用补充：
["精彩集锦", "深度解析", "争议"]

足球动作标签描述场上发生了什么（进球/射门/扑救等），足球分析标签描述解说在做什么类型的点评（战术拆解/历史纪录/裁判争议等）。每个时刻应同时包含动作 + 分析/补充标签，共 3-5 个。允许使用列表外的自定义中文标签。

### Sports Tag Decision Rules（关键——体育内容必须遵守）：

**Step 1：选标签之前，先确定结果。**
对每个动作时刻，问自己："这个动作的最终结果是什么？"
- 球打进了球门 → 必须包含"进球"和"射门"两个标签（进球是射门的子集）
- 球没进（被扑/偏出/中柱）→ 标签是"射门"或"扑救"，不要加"进球"
- "射门"是所有朝球门打的动作的统称，无论结果如何都应标注

**Step 2：分类规则表：**

| 情况 | 必须使用的主标签 | 信号词举例 |
|------|----------------|-----------|
| 进球 | **进球** + 射门 | 破门、进球、得分、入网、扳平、反超、领先、比分X-X |
| 射门未进 | 射门 | 射偏、打高、被扑、中柱、险些、差一点、没能打进 |
| 门将扑救 | 扑救 | 扑出、化解、门将出击、神扑 |
| 点球命中 | 进球 + 射门 + 点球 | 点球命中、罚进、一蹴而就 |
| 点球未中 | 射门 + 点球 | 罚失、射偏、被扑出 |
| 乌龙球 | 进球 + 射门 + 乌龙球 | 乌龙、自摆乌龙、打入自家球门 |
| 头球破门 | 进球 + 射门 + 头球 | 头球破门、头球攻门得手 |

**Step 3：处理 ASR 噪声（语音识别错误）：**
语音识别经常把关键中文足球术语识别错：
- "破门" → 可能显示为：破满、泼满、薄面、迫门、破闷
- "进球" → 可能显示为：进求、尽球、金球、禁球
- "扳平" → 可能显示为：半平、板平、般平
- "得分" → 可能显示为：的分、德分

当看到语音上类似进球确认词的乱码文本，且上下文指示有进球事件（解说兴奋、回放、比分变化），分类为"进球"。

**Step 4：用上下文线索确认进球：**
即使没有明确的"破门/进球"词汇，以下信号可确认进球：
- 射门后解说突然变得兴奋/庆祝性语调
- 提到比分变化（"比分变成X比X"）
- 回放讨论进球过程
- 提到"扳平/反超/领先/绝杀/绝平"
- 球员庆祝或观众爆发

**但需要警惕 ASR 噪声导致的误判：**
- 如果"破门"后紧接着出现"打在人墙上"、"被扑"、"偏出"等否定词，说明实际没有进球
- 解说可能先说"任意球破门"表达能力，后面又说"打在了XX身上"表示实际没进——以最终结果为准
- 关键判断：看后续 30 秒内是否有比分变化确认。如果没有比分变化，很可能不是进球
- **赛后总结不是进球事件**：如果是全场结束后的总结性回顾（"最终X比X"、"全场比赛结束"），event_type 应为 "analysis"

**⚠️ 回放解说 ≠ 赛后总结：**
- 很多视频是「球迷/解说看比赛回放」的形式——解说一边看回放一边分析进球过程（"这个球落点太棒了"、"没盯住XX"、"一锤定音"）。这种情况下回放讨论本身就是进球事件的唯一呈现方式，**必须标为 goal**，不能降为 analysis
- 判断标准：如果解说讨论的内容指向一个明确的进球结果（比分变化、"破门/进球/一锤定音"、球员名+进球动作），且该进球尚未被其他时刻标注过，就应该标为 goal
- 简言之：**event_type 优先级：goal > shot > save > card > analysis**。当一个片段同时包含进球事实和战术分析时，event_type 选 goal，战术分析内容放入 tags

**绝不将已确认的进球仅标为"射门"。进球时必须同时包含"进球"和"射门"两个标签。**

**Step 5：避免重复标注同一进球：**
一个进球可能产生多个转录片段（直播呼叫→回放→分析→庆祝）。这些都是同一个进球事件。规则：
- 每个实际进球只创建一个标为"进球"的时刻——选择最完整捕获动作的片段（铺垫→射门→确认）
- 同一进球的**后续重复讨论**（完全重复的内容，无新信息）应标为"战术亮点"或"赛事高光"
- 判断方法：同一射手、时间戳相隔 2-3 分钟内、解说提到"刚才的进球/这个球/这脚射门"且无新的回放画面
- **注意**：如果每个进球在转录中只出现一次（如回放解说型视频每个进球只有一段讨论），那每段都必须标为 goal，不能因为"像分析"就降级

### 足球分析 Tag 判定规则：

| 标签 | 使用场景 | 判定依据 |
|------|---------|---------|
| 战术解析 | 解说拆解战术细节 | 提及阵型/跑位/传球路线/防守站位 |
| 球星表现 | 聚焦个人发挥 | 提及球员名字+具体表现评价 |
| 历史纪录 | 涉及纪录或里程碑 | 追平/打破纪录、进球数统计 |
| 裁判争议 | 判罚争议 | 对裁判判罚的质疑或慢镜分析 |
| 数据分析 | 引用数据 | 控球率/跑动距离/射正数等统计 |
| 教练布置 | 教练决策 | 换人意图、战术调整、临场指挥 |
| 阵型变化 | 阵型调整 | 从X-X-X变为Y-Y-Y、攻防转换 |
| 体能管理 | 体能话题 | 跑动下降、体能分配、加时赛体能 |
| 赛前分析 | 赛前前瞻 | 首发预测、对位分析、历史交锋 |
| 赛后总结 | 赛后评价 | 全场总结、MVP评选、赛后数据 |
| 经典对决 | 巨星对决 | 两位核心球员正面对抗 |
| 名嘴金句 | 解说名言 | 经典点评、犀利吐槽、专业金句 |
